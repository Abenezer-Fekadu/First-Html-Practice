Does It Matter to Use Relative Size

Nowdays as browsers features are advanced and matured, we had to consider how to work with those new features when designing and developing our sites. These featueres allowed us to create a robust designs. One example of this feature is browser zooming, which allowed us users to zoom everything on the page by scaling up and down proportionally depending on our preferences. But as browsers handle page zoom for us, there is a difference in how the the UI appears when the page is zoomed depending on the weather the default browser size has been adjested as well. Here is why it is usefull to use relative css units and unitless values like (%, em, rem) more consistently, in our css to meet the needs of ever-changing viewport sizes. Since giving units in this way able us to zoom the viewport without loss of the content and able us to resize text without being forced to scroll both horizontally and vertically to read the content, I prefer relative sizing method.

The other main thing we want when we are designing a website is its flexibility, like how the site response with the browsers scale changing and still having maximum readability. So I say using relative size does matter in order to make our site more attractive and user friendly.Such as (%, em, rem) beacause this units provide flexibility and the ability to scale up and down, Instead of being stuck with fixed size. This units are scalable units which are translated by the browser into pixel values,in this case they can be translated in the browser as anything from the 16px which is default browser width to any other value depending on the font size on the design. Which makes them best for responsive design. On the other hand if we are using px unit, it will stay explicitly fixed in size bacause the browser displays it exactly as it is.

CSS3 Changes

CSS3 is the latest evolution of the Cascading Style Sheets language and aims at extending CSS2. It brings a lot of new features and additions, like rounded corners, shadows, gradients, transitions or animations, as well as new layouts like multi-columns, flexible box or grid layouts.

CSS3 is backword compatible with CSS2. It will not make any code written in CSS2 as invalid. it makes the look and feel of a webpage even better. They load faster and the time required to build a page is even less.

In CSS3, there’s no need to include an element for every background image; it provides us with the ability to add more than one background image to any element, even to pseudo-elements.

Box-sizing allows developers to get the sizing of elements right without having to subtract dimensions for padding and borders. With the box-sizing property, the padding and border are included in the height and width.

CSS3 brings with it support for some new ways of describing colours. The most notable change with these new color types is the ability to declare semitransparent colors, this due to the options: HSL, HSLA, and RGBA. RGBA

RGBA works just like RGB, except that it adds a fourth value: alpha, the opacity level or alpha transparency level. The first three values still represent red, green, and blue. For the alpha value, 1 means fully opaque, 0 is fully transparent, and 0.5 is 50% opaque. You can use any number between 0 and 1 inclusively.
HSL stands for hue, saturation, and lightness. With HSL you can tweak either just the saturation or the lightness while keeping the same base hue.
Opacity sets the opaqueness of the element on which it’s declared, similar to alpha.

These effects allow elements on a page to rotate, grow, shrink, flip, or translate into a different color. For the first time, elements created in CSS can move on screen without requiring any JavaScript or Flash code. With transitions, an element can seamlessly change size and color. You can set a duration for a transition, e.g. creating a button that slowly expands and changes color when you mouse over it.

Box-sizing allows developers to get the sizing of elements right without having to subtract dimensions for padding and borders. With the box-sizing property, the padding and border are included in the height and width.

The border-radius property lets you create rounded corners without the need for images or additional markup. If we had wanted different-sized corners, we could declare up to four unique values.

It provides the ability to add drop shadows to elements using the box-shadow property. This property lets you specify the color, height, width, blur, and offset of one or multiple inner and/or outer drop shadows on your elements.

The linear-gradient() function sets a linear gradient as the background image. To create a linear gradient you must define at least two color stops. Color stops are the colors you want to render smooth transitions among. You can also set a starting point and a direction (or an angle) along with the gradient effect.

Radial gradients are circular or elliptical gradients. Rather than proceeding along a straight axis, colors blend out from a starting point in all directions.

Developers can edit elements by name, class, type, attribute, and more.

Text-shadow adds shadows to individual characters in text nodes.
