Does It Matter to Use Relative Size

Nowdays as browsers features are advanced and matured, we had to consider how to work with those new features when designing and developing our sites. These featueres allowed us to create a robust designs. One example of this feature is browser zooming, which allowed us users to zoom everything on the page by scaling up and down proportionally depending on our preferences. But as browsers handle page zoom for us, there is a difference in how the the UI appears when the page is zoomed depending on the weather the default browser size has been adjested as well. Here is why it is usefull to use relative css units and unitless values like (%,em, rem) more consistently, in our css to meet the needs of ever-changing viewport sizes. Since giving units in this way able us to zoom the viewport without loss of the content and able us to resize text without being forced to scroll both horizontally and vertically to read the content, I prefer relative sizing method.

The other main thing we want when we are designing a website is its flexibility, like how the site response with the browsers scale changing and still having maximum readability. So I say using relative size does matter in order to make our site more attractive and user friendly.Such as (%, em, rem) beacause this units provide flexibility and the ability to scale up and down, Instead of being stuck with fixed size. This units are scalable units which are translated by the browser into pixel values,in this case they can be translated in the browser as anything from the 16px which is default browser width to any other value depending on the font size on the design. Which makes them best for responsive design. On the other hand if we are using px unit, it will stay explicitly fixed in size bacause the browser displays it exactly as it is.
